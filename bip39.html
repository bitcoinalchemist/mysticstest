I Ching × BIP39

Binary encoding of the Book of Changes for Bitcoin mnemonics

Encoding Example Reference

Each BIP39 word (index 0–2047) maps to a pair of hexagrams via its 11-bit binary index: upper 6 bits → primary gua (0–63), lower 5 bits → secondary gua (0–31). A 12-word seed becomes 24 hexagrams.

Encoding

A single hexagram carries 6 bits. Two hexagrams carry 12 bits — one more than the 11 bits required to index any BIP39 word. The secondary hexagram's top line is always fixed to 0, giving 6 + 5 = 11 usable bits exactly.

Bits are read left→right = lines 1→6 (bottom→top), so the rightmost bit maps to the top line.

Encode: index → 11-bit binary → split 6|5 → look up both hexagrams.
Decode: index = primary × 32 + secondary.
